# Streamlit UI Module

This module provides a web-based user interface for interacting with the Indic-Law-Assistant. It allows users to ask legal questions, receive answers generated by the RAG pipeline, and manage their conversation history.

The primary application is `streamlit_app.py`.

## Features (streamlit_app.py)

-   **User Authentication**: Secure login with username and password, interacting with the backend API's token authentication. Supports token refresh.
-   **Interactive Chat Interface**: Users can type questions and receive streamed responses from the backend.
-   **Streaming Responses**: Answers from the RAG pipeline are displayed in real-time as they are generated.
-   **Chat History**: Maintains a history of the current conversation.
-   **Language Selection**: Users can choose the language for interaction (English or Malayalam), which is then passed to the backend API to handle translation if configured.
-   **Clear Conversation**: Option to clear the current chat history.
-   **Custom Styling**: Enhanced user interface with custom CSS for better readability and user experience.

## Key Files

-   **`streamlit_app.py`**: This is the main Streamlit application. It handles:
    -   User login and session management (JWT tokens).
    -   Sending user queries to the FastAPI backend (`/stream` endpoint).
    -   Receiving and displaying streamed responses.
    -   Managing chat history and language preferences in the session state.
    -   Rendering the user interface components.
-   **`main.py`**: This appears to be an older or alternative version of the UI. It communicates *directly* with an Ollama API endpoint (`http://localhost:11434/api/generate`) rather than the project's FastAPI backend. It has a simpler chat implementation and does not include features like user authentication or language selection that integrate with the project's backend. **For typical use with the Indic-Law-Assistant backend, `streamlit_app.py` should be used.**
-   **`requirements.txt`**: Lists Python dependencies specific to this Streamlit UI module (e.g., `streamlit`, `requests`, `python-dotenv`, `httpx`).
-   **`streamlit_app.log`**: Log file generated by `streamlit_app.py` to record events and errors.

## Setup and Execution

The main `README.md` in the root of the project contains detailed setup instructions for the entire application. To run the Streamlit UI specifically:

1.  **Ensure Backend API is Running**: The `streamlit_app.py` UI requires the FastAPI backend (from the `api` module) to be running, as it makes requests to `http://localhost:8000`.
2.  **Navigate to the Streamlit UI folder**:
    ```bash
    cd streamlit_ui/
    ```
3.  **Install dependencies**:
    ```bash
    pip install -r requirements.txt
    ```
    *(Note: Ensure that the backend API's dependencies are also installed and the API is running.)*
4.  **Environment Variables**: `streamlit_app.py` uses `python-dotenv` to load environment variables. While not explicitly shown with a `.env` file in the provided code, it's good practice if API URLs or other settings need to be configurable. The backend API URL is currently hardcoded as `http://localhost:8000`.
5.  **Run the Streamlit app**:
    ```bash
    streamlit run streamlit_app.py
    ```
    The application should open in your default web browser.

## Usage

1.  Open the Streamlit application in your browser.
2.  Log in using the credentials configured in the backend API (see `api/auth.py` for default users or how authentication is handled).
3.  Select your preferred language from the sidebar.
4.  Type your legal question into the input box and press "Send Message" or Enter.
5.  The assistant's response will be streamed into the chat window.
6.  You can clear the conversation using the button in the sidebar.

## Note on `main.py`

If you wish to run the simpler `main.py` (which directly contacts Ollama and does not use the project's FastAPI backend):

1.  Ensure Ollama is running and the model specified in `main.py` (e.g., `deepseek-r1:1.5b`) is available.
2.  Run:
    ```bash
    streamlit run main.py
    ```
    This version does not require the FastAPI backend to be running but will also not have the advanced features or context from the RAG pipeline.
